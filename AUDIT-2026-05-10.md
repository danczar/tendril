## 1. Summary — top 5 highest-impact issues

1. **`cargo fmt --check` fails across the workspace** — recently-edited bridge.rs and main.rs are unformatted; CI has no fmt gate, so this drifts every commit. Hours of bikeshedding waiting to happen.
2. **Sync filesystem calls inside async fns block the tokio runtime** — every `std::fs::*` call in async paths (orchestrator, deps, audio convert, ffmpeg/ytdlp ensure) parks a worker thread. For the multi-MB Python tarball extract this is mitigated by `spawn_blocking`, but per-file calls aren't.
3. **`ytdlp_binary_name` / `ffmpeg_binary_name` defined three times each** ([deps/mod.rs:404](crates/tendril-core/src/deps/mod.rs:404), [deps/status.rs:101](crates/tendril-core/src/deps/status.rs:101), [bridge.rs:903](crates/tendril-ui/src/bridge.rs:903)). [bridge.rs:912](crates/tendril-ui/src/bridge.rs:912) `resolve_ffmpeg` also re-implements `deps::ffmpeg::find_on_path`. Already drifted once during the dep-manager refactor — high churn risk.
4. **YouTube search swallows upstream errors** — [youtube/search.rs:34-41](crates/tendril-core/src/youtube/search.rs:34) does `tracks.into_iter().flat_map(|r| r.items.items)` which silently drops `Err` variants. A failed RustyPipe call returns `Ok(vec![])` to the UI.
5. **CI runs zero quality gates** — [.github/workflows/release.yml](.github/workflows/release.yml) only builds on tag push. No clippy, no fmt, no tests on PR/push to main. Combined with `cargo fmt --check` already failing, regressions are guaranteed.

## 2. Findings by severity

### Critical
*None.* The codebase builds cleanly (0 build warnings), all 9 tests pass, no `unsafe` UB, no obvious panics in hot paths.

### High

**H1.** [crates/tendril-core/src/audio/convert.rs:35](crates/tendril-core/src/audio/convert.rs:35), [:75](crates/tendril-core/src/audio/convert.rs:75) — `std::fs::copy` inside `async fn`
- **Why:** Blocks the tokio worker for the duration of the copy. WAV passthrough on a 50 MB file stalls the runtime.
- **Fix:** `tokio::fs::copy(...).await` (already in scope via tokio).

**H2.** [crates/tendril-core/src/pipeline/orchestrator.rs:126](crates/tendril-core/src/pipeline/orchestrator.rs:126), and the same pattern across [deps/demucs_bundle.rs:92](crates/tendril-core/src/deps/demucs_bundle.rs:92), [:150](crates/tendril-core/src/deps/demucs_bundle.rs:150), [deps/ffmpeg.rs:57](crates/tendril-core/src/deps/ffmpeg.rs:57), [:189](crates/tendril-core/src/deps/ffmpeg.rs:189), [:266](crates/tendril-core/src/deps/ffmpeg.rs:266), [:271](crates/tendril-core/src/deps/ffmpeg.rs:271), [deps/ytdlp.rs:28](crates/tendril-core/src/deps/ytdlp.rs:28), [:67](crates/tendril-core/src/deps/ytdlp.rs:67), [:72](crates/tendril-core/src/deps/ytdlp.rs:72), [splitter/engine.rs:42](crates/tendril-core/src/splitter/engine.rs:42), [:45](crates/tendril-core/src/splitter/engine.rs:45), [orchestrator.rs:212](crates/tendril-core/src/pipeline/orchestrator.rs:212), [:219](crates/tendril-core/src/pipeline/orchestrator.rs:219) — sync `std::fs::*` in async contexts.
- **Why:** Each call parks a tokio worker. Most are tiny (mkdir, rename), but they add up and contradict the rest of the file's async-ness.
- **Fix:** Convert to `tokio::fs::*` where the function is already async; for atomic per-file ops use `tokio::task::spawn_blocking` once around the whole batch.
- **Verify:** Pure perf, no correctness issue if traffic is light.

**H3.** [.github/workflows/release.yml](.github/workflows/release.yml) — CI is release-only.
- **Why:** No clippy, no tests, no fmt on PR/push. README's "everything green" promise is hand-waved.
- **Fix:** Add a `ci.yml` running on push/PR with `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, `cargo test --workspace`. Stable + macOS + Windows matrix.

**H4.** [crates/tendril-core/src/youtube/search.rs:34-41](crates/tendril-core/src/youtube/search.rs:34) — search errors silently swallowed.
- **Why:** `tokio::join!` returns `(Result, Result)`. `tracks.into_iter().flat_map(|r| r.items.items)` treats `Err` as an empty iterator. UI never sees rate-limits, 5xx, parse failures. Dep-manager-quirk déjà vu.
- **Fix:**
  ```rust
  let (tracks, videos) = tokio::join!(...);
  if let (Err(t), Err(v)) = (&tracks, &videos) {
      return Err(YoutubeError::Search(format!("both tabs failed: tracks={t} videos={v}")));
  }
  let track_items = tracks.map(|r| r.items.items).unwrap_or_default();
  let video_items = videos.map(|r| r.items.items).unwrap_or_default();
  ```

**H5.** Triplicated binary-name helpers — [deps/mod.rs:404-411](crates/tendril-core/src/deps/mod.rs:404), [deps/status.rs:101-108](crates/tendril-core/src/deps/status.rs:101), [bridge.rs:903-910](crates/tendril-ui/src/bridge.rs:903), [bridge.rs:931-938](crates/tendril-ui/src/bridge.rs:931). Plus [bridge.rs:912 `resolve_ffmpeg`](crates/tendril-ui/src/bridge.rs:912) re-implements [deps/ffmpeg.rs:282 `find_on_path`](crates/tendril-core/src/deps/ffmpeg.rs:282).
- **Why:** Already drifted: bridge.rs's `resolve_ffmpeg` doesn't honor the platform-specific Windows-managed-first priority that `deps::ffmpeg::ensure` does. So the pipeline's actual ffmpeg path can disagree with what's recorded in versions.json on Windows.
- **Fix:** Promote one canonical pair (`pub fn ytdlp_binary_name() -> &'static str` + ditto ffmpeg) in `deps/mod.rs`, re-export from `deps/`. Make `resolve_ffmpeg` call `deps::ffmpeg::ensure` (or a sync `resolve_ffmpeg_now`) so it shares the priority logic.

**H6.** [crates/tendril-core/Cargo.toml:16](crates/tendril-core/Cargo.toml:16) — `regex = "1"` declared crate-locally, ignoring the workspace declaration.
- **Why:** Workspace `Cargo.toml:18` already has `regex = "1"`. The duplicated specifier means version bumps must touch both files. Same pattern works for every other shared dep.
- **Fix:** `regex = { workspace = true }`.

**H7.** No `[profile.release]` tuning — Cargo.toml has no profile section.
- **Why:** Default release leaves debug symbols, no LTO, 16 codegen units. The macOS bundle ships a multi-hundred-MB universal binary; LTO + strip would meaningfully reduce that, and LTO speeds up Slint render hot paths.
- **Fix:** Add to workspace Cargo.toml:
  ```toml
  [profile.release]
  lto = "thin"
  codegen-units = 1
  strip = "symbols"
  ```

**M5.** [crates/tendril-core/src/pipeline/orchestrator.rs:131-153](crates/tendril-core/src/pipeline/orchestrator.rs:131) — 4-stem conversion is sequential.
- **Why:** Each ffmpeg invocation is single-threaded for typical codecs; 4 stems × ~3-5s each = 12-20s wall time. Modern CPUs can run 4 ffmpegs in parallel with negligible memory cost.
- **Fix:**
  ```rust
  use futures::future::try_join_all;
  let convert_futs = stem_paths.iter().map(|p| convert(&ctx.ffmpeg_bin, p, ctx.output_format, &final_dir));
  try_join_all(convert_futs).await.map_err(...)?;
  ```
  Note: this loses per-stem progress events; either keep the old sequential path behind a flag or emit "Converting (parallel)..." once.

**M6.** [crates/tendril-core/src/audio/convert.rs](crates/tendril-core/src/audio/convert.rs) — `convert` and `convert_to` are 95% duplicates.
- **Fix:** Keep `convert_to` (explicit destination) as the workhorse; have `convert` compute the destination then delegate.

**M7.** [crates/tendril-core/src/pipeline/orchestrator.rs:201](crates/tendril-core/src/pipeline/orchestrator.rs:201) — `cancel_rx: &mut watch::Receiver<bool>` but `borrow()` doesn't need `&mut`. Clippy flag.
- **Fix:** Take `&watch::Receiver<bool>`. Caller adjustments trivial.

**M8.** [crates/tendril-ui/src/bridge.rs:122,136](crates/tendril-ui/src/bridge.rs:122) — `tokio::time::sleep(Duration::from_secs(5))` after job success/failure blocks the queue runner.
- **Why:** A 10-job batch waits 50s of pure dead time so the user can read "Done" / "Failed".
- **Fix:** UX-driven; either drop to 1-2s or detach the success-display from the runner (UI fades out on next tick).

**M9.** [crates/tendril-core/src/dirs.rs:20](crates/tendril-core/src/dirs.rs:20), [:70](crates/tendril-core/src/dirs.rs:70) — `.expect("home directory must exist")` panics on systems without HOME.

- **Fix:** `.ok_or_else(|| io::Error::new(io::ErrorKind::NotFound, "no home directory"))?`

**M10.** [crates/tendril-ui/src/main.rs:153](crates/tendril-ui/src/main.rs:153) — sole `unsafe` block has no `// SAFETY:` comment.

**M11.** [crates/tendril-core/src/deps/versions.rs:15-19](crates/tendril-core/src/deps/versions.rs:15) — `impl Default for FfmpegSource` is hand-written.

**M12.** [crates/tendril-core/src/youtube/search.rs:37,41](crates/tendril-core/src/youtube/search.rs:37) — collects to `Vec<_>` then `.into_iter()`. Wasted allocation.

**M13.** [crates/tendril-ui/src/models.rs:90](crates/tendril-ui/src/models.rs:90) — `job.id as i32` (id is u64).
